聊天備份

聊天訊息主要存放在 TIM 之中。
因為軟體的設計缺陷，使得聊天訊息常有遺失。
官方内建的聊天備份工具可以説形同虛設，新版甚至刪除了這個功能。

個人過往慣用的方式是利用舊版可 Ctrl + A 的方式全選記錄，而後存檔為 Doc + Pdf 。
不過正如前文所言，因為軟體的設計缺陷，我需要後期進行詳細校對。
因為需要經歷手工重排版，校對的時間和效率還是一言難盡的。

我會使用 新的貼文 對該方式進行詳細解説。

系統重灌

漫長的準備后，開始重灌系統。
該貼文會停止更新，新系統的一切進入正軌會再次編輯。
……
現在，一切已進入正軌。

製作恢復優盤

  1. 下載 R7000 2020 恢復鏡像 ；
  2. 格式化隨身碟的系統為 FAT32 ，標簽為 RECOVERY；
  3. 將恢復鏡像解壓至隨身碟根路徑，恢復隨身碟製作完成。

開始重灌系統

  1. 設定 - 更新與安全性 - 復原；
  2. 進階啟動 - 重新啟動，等待重新啟動；
  3. 疑難排解 - 啟動設定，選擇恢復隨身碟；
  4. 如果不顯示隨身碟，進行熱插拔；
  5. 進入恢復隨身碟系統，按照指引安裝。

系統重灌的過程中會伴隨多次自動重啟，耐心等待即可。
若出現異常的提示信息，請耐心等待，若無反應，使用 Alt +G 跳過即可。
不知為何，這次重灌我沒出現異常提示信息……

恢復賬戶資料

  1. 使用離線狀態進入系統，設定用戶名為 Magstic；
  2. 使用 Edge Block 禁用 Edge；
  3. 安裝 360資料夾 和 Start11，配置輸出規則為封鎖連線；
  4. 安裝 Mihomo Party 和 Chrome，並完善配置；
  5. 在磁碟管理中分配 Archive 為 D，賦予 180 GB 的空間；
  6. 使用 更新工具 更新 Windows 系統為 22H2；
  7. 使用設定内建的 Windows Update 修補更新；
  8. 使用 Zulu 8 替代 Java 8；
  9. 下載 QQNT ，拷貝 Msg3.0.db 至 C:\Users\Magstic\Documents\Tencent Files\XXXXXX，在訊息窗使用滑鼠中鍵向上快速滾動以恢復聊天；
  10. 使用 Geek 卸載系統自帶的 愛奇藝、WPS、騰訊視頻，并檢查 Registry ；
  11. 配置 Hexo 環境，拷貝 node_modules source scaffolds _config.butterfly.yml _config.yml package.json並覆蓋；
  12. 安裝其他軟體，配置到此結束。

茶後小敘

這次的系統重灌完美結束。
唯一的缺憾，是 QQ 訊息需要我手動去翻才可讀取……
説真的，DC 的雲端存儲，以及 TG 的閲後即焚，QQ 是一個都沒學習到位。

Zulu 8 取代 Java 8 是臨時起意，沒想到其效能確實强大。
Minecraft 1.12.2 冗長的模組加載時間，在 Zulu 8 下有明顯的改善。

最後順便吐槽，微軟的伺服器真的好慢哦XD
